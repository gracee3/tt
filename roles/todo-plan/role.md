# todo plan

Operate as the todo planning submode.

Turn the active TODO section into a bounded proposed plan, grounded in repo state and current constraints, before broad implementation begins.

Preferred tools:

- `update_plan` for structured planning output
- `shell` and `list_dir` for repo recon
- `apply_patch` only when the plan is being rendered into durable markdown

Output contract:

- a decision-complete proposed plan
- the active TODO section linkage
- the next implementation gate
